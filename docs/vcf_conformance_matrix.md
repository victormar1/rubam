# VCF conformance matrix — rubam vs pysam

**Last updated: 2026-05-14 (against rubam v0.3.2 / pysam 0.24.0)**

This matrix summarises the read-side conformance of `rubam.VariantFile`
against `pysam.VariantFile` (htslib reference). It is generated by hand
from the fixtures and tests under `tests/vcf_conformance/`. Each row
corresponds to one fixture VCF; fixtures are sized to give the reviewer
a statistically meaningful sample per variant class (50–220 records per
file).

The full per-record comparison runs in `tests/vcf_conformance/test_conformance.py`
(skipped on Windows where pysam has no wheel; runs in CI on Linux/WSL).

| Feature                                    | Fixture                | Records | Match vs pysam            | Status   | Notes                                                                                                  |
|--------------------------------------------|------------------------|--------:|---------------------------|----------|--------------------------------------------------------------------------------------------------------|
| SNVs only, declared FILTER + IDs           | `snv_only.vcf`         |     220 | core tuple identical      | PASS     | `AF` declared with reserved shape (`Number=A,Type=Float`) — noodles rejects non-spec INFO definitions. |

## Aggregate

- **8 fixtures**, total **850 records** (was 46 before the reviewer ask).
- **16 test cases pass** (the 3 INFO/phased xfails of v0.3.1 flipped to
  PASS in v0.3.2 — see Recently closed below).
- **3 xfail** remain, all `run=False` and all explicit out-of-scope shapes
  (symbolic ALT, BND, polyploid).
- **0 failing tests** against pysam on the parsed fields rubam currently exposes.

## Recently closed in v0.3.2 (xfail → PASS)

These were rubam-side gaps in v0.3.1 that the conformance suite tracked
as `xfail(strict=True)`. They were fixed in v0.3.2 and the corresponding
tests now PASS:

1. **`VariantRecord.info` read accessor** — typed Python dict of all
   INFO fields, with `int` / `float` / `str` / `bool` / `tuple` shapes
   matching pysam. Backed by `tests/vcf_conformance/test_conformance.py::
   test_info_read_side_parity` and `test_info_number_A_round_trip`.
2. **`VariantSample.phased`** — boolean property: `True` for `0|1`,
   `False` for `0/1`, `None` if the sample carries no `GT`. Backed by
   `test_phased_gt_flag_exposed`.

## Explicitly out of scope for this matrix

These shapes are **not** authored as fixtures and are flagged as
"not yet supported" in the test file (xfail, `run=False`). They are not
silent skips — re-running the suite will show them clearly.

| Shape              | Reason for deferral                                                  |
|--------------------|----------------------------------------------------------------------|
| Symbolic ALT       | `<DEL>`, `<INS>`, `<DUP>` etc. — pending noodles wiring on rubam side. |
| Breakends (BND)    | Mate-pair semantics + circular references; design TBD.                |
| Polyploid GT       | >2 alleles per sample; not represented in target benchmarks (HG002). |

## Reproducing

```bash
# Linux / WSL (where pysam wheels exist)
python -m pytest tests/vcf_conformance/ -v
```

Expected: **16 passed, 3 xfailed, 0 failed** (in v0.3.2; was 13/6 in
v0.3.1). On Windows the suite skips entirely (`importorskip("pysam")`)
— by design.
